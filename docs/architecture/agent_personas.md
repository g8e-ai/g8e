# Agent Persona System

Last Updated: 2026-05-10
Version: v0.2.2

## Overview

The g8e platform utilizes a centralized agent persona system to define AI identities, roles, and behavioral constraints. This system ensures consistency across Python (`g8ee`) and Node.js (`g8ed`) components while optimizing for prefix-cache performance through a standardized XML-tagged prompt structure.

### The Persona Catalog

| Agent | Code ID | Role | Tier | Purpose |
|---|---|---|---|---|
| **Triage** | `triage` | `classifier` | `lite` | Initial classification of complexity, intent, and posture. |
| **Sage** | `sage` | `reasoner` | `primary` | Senior architect for complex investigative tool-loops. |
| **Dash** | `dash` | `responder` | `assistant` | Fast-path responder for simple, single-turn tasks. |
| **The Tribunal** | `tribunal` | `arbitrator` | `lite` | Collective aggregate of the command-translation ensemble. |
| **Auditor** | `auditor` | `auditor` | `primary` | Final verification gate for Tribunal candidates. |
| **Warden** | `warden` | `defender` | `lite` | Defensive coordinator for pre-execution risk analysis. |
| **Scribe** | `scribe` | `summarizer` | `lite` | Generates concise (3-7 word) case titles. |
| **Codex** | `codex` | `analyzer` | `lite` | Extracts durable user preferences and scrubbed summaries. |
| **Judge** | `judge` | `evaluator` | `primary` | Dispassionate evaluator for gold-standard benchmarking. |

## Authority & Registry

The canonical truth for agent personas resides in **Python models** located in `@/components/g8ee/app/models/personas/`. These models enforce field presence and alignment via Pydantic and provide structured system prompt generation.

- **Authoritative Source**: `@/components/g8ee/app/models/personas/` (Python classes).
- **Derivative JSON**: `@/shared/constants/agents.json` (Generated by `scripts/data/sync-personas.py` for Node.js consumption).
- **Metadata Fields**:
    - `id`: Unique identifier (e.g., `triage`, `sage`).
    - `display_name`: Human-readable name.
    - `icon`: Material Design icon name.
    - `description`: Brief role summary.
    - `role`: Behavioral category (e.g., `classifier`, `reasoner`).
    - `model_tier`: Runtime routing target (`primary`, `assistant`, `lite`).
    - `tools`: List of permitted tool names.
    - `identity`: Core personality and behavioral instructions.
    - `purpose`: Technical objectives and pipeline role.
    - `autonomy`: Directive prose affirming the agent's agency.
    - `output_contract`: (Optional) Strict wire format definition (e.g., Tribunal members).

## Core Pipeline Agents

### 1. Triage (The Gatekeeper)
- **Code Reference**: `@/components/g8ee/app/models/personas/triage.py`
- **Purpose**: First-turn classification of complexity, intent, and user posture.
- **Complexity Calibration**: 
    - `simple`: Routine inquiries or single-step tasks requiring no novel reasoning.
    - `complex`: Multi-step operations, ambiguity, or security-sensitive paths.
- **Posture Analysis**: Gauges `normal`, `escalated`, `adversarial`, or `confused` mindset. 
- **Invariants**: First-turn messages CANNOT be `adversarial`. Security-related keywords ALWAYS force `complex`.

### 2. Sage (Reasoning Authority)
- **Code Reference**: `@/components/g8ee/app/models/personas/sage.py`
- **Purpose**: Architect of complex investigations. Drives multi-step tool loops.
- **Intent Articulation**: Describes functional goals (e.g., "Determine if nginx errors started before the 14:20 deploy") without naming shell tools.
- **Interrogation Protocol**: If ambiguity persists, issues exactly three binary (YES/NO) questions to the user.

### 3. Dash (Fast-Path Responder)
- **Code Reference**: `@/components/g8ee/app/models/personas/dash.py`
- **Purpose**: Resolves straightforward requests with minimal latency.
- **Escalation**: Hands off to **Sage** if multi-step planning or deep reasoning is required.

## The Tribunal & Safety

### 4. The Tribunal (Command Translation)
A five-member ensemble translating Sage's intent into executable commands through ideological consensus. Members are isolated to ensure diverse technical viewpoints.

| Member | Code ID | Lens | Focus |
|---|---|---|---|
| **Axiom** | `axiom` | `composition` | Elegant, efficient pipeline composition. |
| **Concord** | `concord` | `safety` | Defensive discipline and minimal risk. |
| **Variance** | `variance` | `edge_cases` | Robustness against spaces, symlinks, and null inputs. |
| **Pragma** | `pragma` | `convention` | Idiomatic patterns (e.g., `ss` vs `netstat`). |
| **Nemesis** | `nemesis` | `adversary` | Plausible flaws to stress-test the Auditor. |

### 5. Warden (Defensive Coordination)
Warden is a coordinator (`@/components/g8ee/app/models/personas/warden.py`) that orchestrates specialized sub-agents for pre-execution risk analysis.

- **Sub-Agents**:
    - `warden_command_risk`: Classifies command blast radius (LOW/MEDIUM/HIGH).
    - `warden_file_risk`: Evaluates file operation sensitivity and git-reversibility.
    - `warden_error`: Analyzes failures for `AUTO_FIXABLE` or `ESCALATE`.
- **Stake Reputation**: Sub-agents stake reputation on accuracy. Over-caution or under-caution results in reputational penalties.

### 6. Auditor
- **Code Reference**: `@/components/g8ee/app/models/personas/auditor.py`
- **Purpose**: Final judge of Tribunal candidates. 
- **Responsibility**: Selects the winner (unanimous, majority, or swap) based on intent fidelity and technical scrutiny. The only agent authorized to commit to the global reputation Merkle-root.

## The Information Isolation Principle

Agents operate in a sealed environment to preserve diversity and technical honesty.

| Role | Information Visible | Isolation Goal |
|---|---|---|
| **Triage** | User message + brief history | Prevent routing bias |
| **Sage/Dash** | Triage result + full history | Focus on reasoning/speed |
| **Tribunal** | Sage's intent + OS context | Prevent ideological collision (Amnesia) |
| **Auditor** | Intent + Anonymized Candidates | Converge on technical truth |

## Canonical Prompt Layout

System prompts are constructed via `@/components/g8ee/app/models/personas/base.py` using a standardized XML-tagged pattern to optimize prefix-cache efficiency:

1. `<role>`: The behavioral category.
2. `<output_contract>`: (If applicable) Strict wire format.
3. `<identity>`: Core personality and voice.
4. `<purpose>`: Technical objectives and technical discipline.
5. `<autonomy>`: Agency and accountability directive.

This structure allows the inference engine to cache the shared `<role>` and `<output_contract>` prefixes across different agents, reducing time-to-first-token.

## Reputation & Staking

The platform uses a Phase 3 Reputation System to enforce performance:
- **EMA Updating**: Agent scalars are updated using an Exponential Moving Average (half-life: 50).
- **Slashing**: Catastrophic failures result in tiered retention (e.g., Tier 1: 25% retention).
- **Merkle Commitments**: Auditor's reputation updates are cryptographically bound to a global scoreboard.
