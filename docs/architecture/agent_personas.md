# Agent Persona System

Last Updated: 2026-05-13
Version: v0.3.0

## Overview

The g8e platform utilizes a tiered agent persona system that separates **Substrate Responsibilities** (governance enforcement and execution) from **Application-Layer Reasoning** (intent generation and consensus). 

- **The Substrate (g8eo)**: Acts as the "Sovereign Operator". It is persona-agnostic but fulfills the role of the **Executor** and **Validator**. It enforces the L1/L2/L3 governance hierarchy and executes through the **Substrate Warden**.
- **The Application Layer (g8ee)**: The reference AI reasoning engine. It implements the specialized LLM identities (Personas) that generate, verify, and coordinate intents before they are sent to the substrate.

## Authority & Registry

The g8e platform maintains a strict "Code-First" registry for agent personas.

1. **Canonical Source**: `@/components/g8ee/app/models/personas/` (Python Pydantic classes). These define the prompts, tools, and constraints.
2. **Derivative Constant**: `@/protocol/constants/agents.json`. A JSON representation generated by `scripts/data/sync-personas.py` for cross-component consumption.
3. **Substrate Implementation**: `@/components/g8eo/internal/services/governance/`. Go-based services that enforce the outcomes of these personas (e.g., verifying a Tribunal signature).

### Persona Metadata Schema
Each persona entry in `agents.json` includes:
- `id`: Unique identifier (e.g., `sage`, `axiom`).
- `role`: Behavioral category (e.g., `reasoner`, `tribunal_member`).
- `model_tier`: Runtime routing target (`primary`, `assistant`, `lite`).
- `identity`: Core personality and behavioral instructions.
- `purpose`: Technical objectives and pipeline role.
- `autonomy`: Directive prose affirming the agent's agency.
- `output_contract`: (Tribunal only) Strict wire format definition for candidates.

## Core Pipeline Agents

### 1. Triage (The Gatekeeper)
- **Role**: First-turn classification.
- **Complexity**: Determines if a task is `simple` (handled by Dash) or `complex` (handled by Sage).
- **Posture**: Gauges user mindset (`normal`, `escalated`, `adversarial`, `confused`). 
- **Invariants**: Security-related keywords ALWAYS force `complex`. First-turn messages CANNOT be `adversarial`.

### 2. Sage (The Architect)
- **Role**: Reasoning authority for complex investigations.
- **Intent Articulation**: Sage describes *what* needs to be done (the intent) without specifying *how* (the command). It speaks to the Tribunal as an architect to a builder.
- **Discipline**: Prioritizes reasoning and evidence collection before proposing any mutation.

### 3. Dash (The Fast-Path)
- **Role**: Resolves straightforward requests with minimum viable work.
- **Escalation**: Hands off to Sage if a request requires multi-step planning or deep reasoning.

## Consensus: The Tribunal

The Tribunal is an ensemble of five specialized LLM members isolated to ensure diverse technical viewpoints. They translate Sage's intent into executable commands.

| Member | Code ID | Lens | Focus |
|---|---|---|---|
| **Axiom** | `axiom` | `composition` | Elegant, efficient multi-stage pipeline composition. |
| **Concord** | `concord` | `safety` | Defensive flags, dry-runs, and minimal-risk discipline. |
| **Variance** | `variance` | `edge_cases` | Robustness against spaces, symlinks, and null inputs. |
| **Pragma** | `pragma` | `convention` | Idiomatic patterns for the specific OS and shell. |
| **Nemesis** | `nemesis` | `adversary` | Proposes plausible-but-flawed candidates to stress-test the Auditor. |

### The Auditor
The **Auditor** is the final judge. It evaluates the five candidates against Sage's intent, identifies the most robust implementation, and provides the "Decision" that the Substrate verifies.

## Defensive Analysis: The AI Warden

In the reference application, the **Warden** is an AI coordinator that orchestrates specialized risk analyzers to provide a "safety signal" for the Operator and the human co-validator.

- **Warden (Coordinator)**: Synthesizes risk levels from sub-agents into a consolidated verdict (`LOW`, `MEDIUM`, `HIGH`).
- **Command Risk Analyzer**: Classifies the "blast radius" of shell commands.
- **File Risk Analyzer**: Evaluates the sensitivity and reversibility of file operations.
- **Error Analyzer**: Classifies failures as `AUTO_FIXABLE` or `ESCALATE`.

---

## The Substrate Warden (g8eo)

Distinct from the AI-layer Warden, the **Substrate Warden** in `g8eo` is the final execution stop.

- **Verifiable Execution**: It performs the actual mutation in an isolated environment.
- **State Integrity**: It verifies the `state_merkle_root` before execution to prevent stale or replayed transactions.
- **Audit Commitment**: It emits a **Signed Action Receipt**, anchoring the result of the mutation to the host-authoritative ledger.

---

## The Information Isolation Principle

To preserve diversity and technical honesty, agents operate under the **Amnesia Principle**:
- **Tribunal Members** are blind to each other's candidates.
- **The Auditor** receives anonymized candidates to prevent source bias.
- **The Engine** only passes the minimum required context to each stage of the pipeline.

---

## Reputation & Staking

The performance of every agent is tracked via the **Reputation System**:
- **EMA Decay**: Performance scores use Exponential Moving Averages to prioritize recent behavior.
- **Slashing**: Catastrophic failures (e.g., an Auditor approving a destructive command flagged by the AI Warden) result in reputational penalties.
- **Accountability**: Reputation updates are cryptographically committed to a global scoreboard, ensuring agents are held accountable for their "votes" in the governance process.
