# g8e Pub/Sub Channel Specification

Official wire-protocol channel reference for the g8e platform. All inter-component pub/sub communication uses these channel formats.

---

## Channel Format

Per-operator-session channels follow this canonical format:

```
{prefix}:{operator_id}:{operator_session_id}
```

- **prefix** -- Channel type: `cmd`, `results`, `heartbeat`
- **operator_id** -- Unique operator identifier
- **operator_session_id** -- Full session ID generated by g8ed as `operator_session_<ts>_<uuid>`

### Source of Truth

`shared/constants/channels.json` is the single canonical definition. Component-level bindings consume it:

| Component | File | Mechanism |
|-----------|------|-----------|
| g8ee (Python) | `components/g8ee/app/constants/channels.py` | `OperatorChannel` class with typed constructors and parser |
| g8eo (Go) | `components/g8eo/constants/channels.go` | `CmdChannel`, `ResultsChannel`, `HeartbeatChannel` functions |

### Critical Rules

1. **Session ID is used verbatim** -- The `operator_session_id` is the full string `operator_session_<ts>_<uuid>` generated by g8ed. Never add, remove, or re-prefix this value.
2. **Use typed constructors only** -- Never format channel names by hand. Use `OperatorChannel` (g8ee) or `ResultsChannel`/`CmdChannel`/`HeartbeatChannel` (g8eo).
3. **Bounded split for parsing** -- Session IDs may contain `:` characters. Use a bounded split (max 2 splits) to preserve the session ID intact.

## Channel Types

### Command Channels (`cmd:`)

Used to send commands from g8ee to g8eo for execution.

**Format:** `cmd:{operator_id}:{operator_session_id}`

**Example:** `cmd:op-abc123:operator_session_1764000000000_a1b2c3d4-e5f6-7890-abcd-ef1234567890`

### Result Channels (`results:`)

Used to send command execution results from g8eo back to g8ee.

**Format:** `results:{operator_id}:{operator_session_id}`

**Example:** `results:op-abc123:operator_session_1764000000000_a1b2c3d4-e5f6-7890-abcd-ef1234567890`

### Heartbeat Channels (`heartbeat:`)

Used for operator heartbeat signals between g8eo and g8ee.

**Format:** `heartbeat:{operator_id}:{operator_session_id}`

**Example:** `heartbeat:op-abc123:operator_session_1764000000000_a1b2c3d4-e5f6-7890-abcd-ef1234567890`

---

## Platform-wide Broadcast Channels

These channels are used for system-wide broadcasts, not per-operator communication.

| Channel | Purpose |
|---------|---------|
| `g8eo_results` | Global results broadcast (deprecated in favor of per-operator channels) |
| `operator_heartbeats` | Global heartbeat aggregation |
| `sse_events` | Server-Sent Events stream |
| `system_events` | System-level event notifications |

## Implementation Examples

### g8ee (Python)

```python
from app.constants import OperatorChannel

# Construct a results channel
operator_id = "op-abc123"
session_id = "operator_session_1764000000000_a1b2c3d4-e5f6-7890-abcd-ef1234567890"
channel = OperatorChannel.results(operator_id, session_id)
# Result: "results:op-abc123:operator_session_1764000000000_a1b2c3d4-e5f6-7890-abcd-ef1234567890"

# Parse a channel
prefix, op_id, sess_id = OperatorChannel.parse(channel)
# Result: ("results", "op-abc123", "operator_session_1764000000000_a1b2c3d4-e5f6-7890-abcd-ef1234567890")
```

### g8eo (Go)

```go
import "github.com/lateraluslabs/g8e/components/g8eo/constants"

// Construct a results channel
operatorID := "op-abc123"
sessionID := "operator_session_1764000000000_a1b2c3d4-e5f6-7890-abcd-ef1234567890"
channel := constants.ResultsChannel(operatorID, sessionID)
// Result: "results:op-abc123:operator_session_1764000000000_a1b2c3d4-e5f6-7890-abcd-ef1234567890"
```

## Common Pitfalls

### Double Prefix Bug (Phantom) - NOT A REAL BUG

A previous investigation incorrectly claimed that channel names contained duplicated `operator_session_` prefixes. This was a misunderstanding: the literal `operator_session_` is part of the session ID value itself (generated by g8ed), not a duplicated prefix.

**Correct understanding:**
- Session ID format: `operator_session_1764000000000_abc-123-def-456-ghi-789`
- Channel format: `results:op-123:operator_session_1764000000000_abc-123-def-456-ghi-789`
- The `operator_session_` appears exactly once — as part of the session ID value.

### Hand-rolled Channel Formatting (Forbidden)

Never construct channel names by string concatenation:

```python
# WRONG
channel = f"results:{operator_id}:{session_id}"

# RIGHT
channel = OperatorChannel.results(operator_id, session_id)
```

### Unbounded Split Parsing (Forbidden)

Never use an unbounded split that would break session IDs containing `:`:

```python
# WRONG - breaks if session_id contains ":"
parts = channel.split(":")

# RIGHT - bounded split preserves session_id
parts = channel.split(":", 2)  # max 2 splits
```

## Testing

Contract regression tests verify that both components produce identical channel strings with realistic session IDs:

- **g8ee:** `tests/unit/utils/test_constants.py::TestOperatorChannelContractRegression`
- **g8eo:** `constants/channels_test.go::TestChannelContractRegression`
