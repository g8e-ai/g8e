# The v0.2.0 Migration: Protobuf-First Protocol and the Sunset of g8es

Last Updated: 2026-05-10
Version: v0.2.2

*Danny Barbour · May 2026 · [github.com/g8e-ai/g8e](https://github.com/g8e-ai/g8e)*

---

Last week we tagged `v0.2.0`, marking the largest architectural shift since the project's inception. We've moved away from the "simulation-first" Docker-heavy model toward a host-native, Protobuf-first substrate that's faster, safer, and easier to audit.

Here are the three big changes and why we made them.

## 1. The Sunset of g8es

Until now, `g8es` was a separate component responsible for persistence and coordination. It was a useful abstraction during early prototyping, but it introduced unnecessary network hops and complex volume-mounting logic.

As of `v0.2.0`, `g8es` is gone. Its responsibilities have been absorbed into the **Operator listen mode**.

When you run `./g8e platform start`, you're launching the `g8e.operator` binary with the `--listen` flag. It manages its own local SQLite state, blob storage, and the HTTP/WSS control plane. This means the Dashboard (`g8ed`) and Engine (`g8ee`) now talk directly to the host's primary Operator. 

## 2. Protobuf-First: The UniversalEnvelope

JSON was great for the "Dash" era, but it's too loose for a production governance plane. We've migrated the entire wire protocol to Protobuf, centered around the `UniversalEnvelope`.

Every command, result, and audit event is now a serialized Protobuf message. This gives us:
- **Type Safety at the Wire:** The Operator rejects malformed envelopes before they even reach the logic layer.
- **L1 Bedrock via Reflection:** We can now enforce hard technical gates (like forbidden patterns) by reflecting over the typed payload before dispatch.
- **Merkle Commitments:** Every envelope carries state roots and context that are verified cryptographically.

If it's not a valid `UniversalEnvelope`, the Operator fails closed. No more silent migration of stale data.

## 3. Host-Native Execution

We've removed Docker from the core platform's runtime. `g8ed`, `g8ee`, and the local Operator now run as native host processes.

**Why the shift?**
- **Performance:** Sub-millisecond latency for the internal control plane.
- **Security:** No more complex bind-mounts of host secrets into containers. Secrets stay on the host, and processes only see what they need via scoped environment variables or tmpfs paths.
- **Local Sovereignty:** The platform state now lives in a repo-local `.g8e/` directory. It’s transparent, easy to backup, and doesn’t disappear when you `docker system prune`.

Docker is now strictly reserved for the **Evaluation Fleet**—the simulated target nodes that the Operator deploys to during benchmarks and testing.

---

## What’s Next?

With the protocol bedrock settled, we’re focusing on **L2 Tribunal Calibrations** and **FIDO2 Co-validation (L3)** hardening. The goal for `v0.3.0` is to make the "Two-Strike Circuit Breaker" a standard across all agent personas.

If you’re running a version older than `v0.2.0`, you’ll need to perform a `platform reset` to migrate your local state to the new Protobuf-first layout.

The repo is at [github.com/g8e-ai/g8e](https://github.com/g8e-ai/g8e).
