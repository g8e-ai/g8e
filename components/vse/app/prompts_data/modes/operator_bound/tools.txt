<context>
[list_files_and_directories_with_detailed_metadata]
Use instead of `ls`. Returns full stat metadata (type, size, mode, mtime, owner). Use for existence checks — returns error if path missing.
- path: directory to list | max_depth: 0-3 | max_entries: default 100, max 500
- target_operator: REQUIRED when multiple operators bound (hostname, operator_id, or index)

[file_read_on_operator]
Read file content. Always read before updating. Use start_line/end_line/max_lines for large files.
- file_path, justification: REQUIRED
- target_operator: REQUIRED when multiple operators bound

[file_create_on_operator]
Create new file. Fails if exists — use file_write_on_operator to overwrite. Creates parent dirs automatically.
- file_path, content, justification: REQUIRED
- target_operator: REQUIRED when multiple operators bound

[file_write_on_operator]
Replace entire file contents. Backs up by default. Use file_update_on_operator for surgical edits.
- file_path, content, justification: REQUIRED | create_if_missing: false | create_backup: true
- target_operator: REQUIRED when multiple operators bound

[file_update_on_operator]
Surgical find-and-replace. old_content must match exactly. Read file first to copy exact text.
Workflow: read → update → verify
- file_path, old_content, new_content, justification: REQUIRED | create_backup: true
- target_operator: REQUIRED when multiple operators bound

[run_commands_with_operator]
Execute shell commands. Chain read-only commands with `&&`. Single destructive command per call.
- command, justification: REQUIRED | timeout_seconds: 300 | expected_output_lines: for UI sizing
- target_operator: REQUIRED when multiple operators bound (optional for single)

[g8e_web_search]
Query external docs and best practices. Search when: planning, uncertain about any command, version-specific, error lookup, security/CVE.
Use `site:` for trusted sources. Include version numbers. Search early — prevents costly mistakes.
