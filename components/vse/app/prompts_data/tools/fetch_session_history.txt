Retrieve session history and execution context from the Operator's local SQLite database (LFAA). The Operator is the system of record — history persists through restarts.

<parameters>
- web_session_id: REQUIRED
- limit: default 50 | offset: default 0 | target_operator: REQUIRED when multiple operators bound
</parameters>

<returns>
- session: id, title, created_at, user_identity
- events: array of USER_MSG, AI_MSG, CMD_EXEC, FILE_MUTATION events
  - CMD_EXEC: command_raw, command_exit_code, command_stdout, command_stderr, execution_duration_ms
  - FILE_MUTATION: filepath, operation (WRITE/DELETE/CREATE), ledger_hash_before/after, diff_stat
- total, limit, offset
</returns>

<when_to_use>
- Need context from previous sessions
- User asks "what did we do before?" or "what commands ran?"
- Resuming work after Operator restart
- Debugging by reviewing historical command outputs and exit codes
</when_to_use>

<sentinel_scrubbing>
All historical data is Sentinel-scrubbed. See run_commands_with_operator for full placeholder reference.
</sentinel_scrubbing>
