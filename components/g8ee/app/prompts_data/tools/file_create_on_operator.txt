Authorized tool for creating files with known content on the g8e Operator system. Preserves exact formatting, creates parent directories, and is tracked in file history for audit and rollback.

Use for:
- Shell scripts (.sh)
- Configuration files (.conf, .yaml, .json, .env, .ini)
- Cron jobs, systemd units, logrotate configs
- Any new file with known content

<parameters>
- file_path, content, justification: REQUIRED
- target_operator: REQUIRED when multiple operators bound
</parameters>

<behavior>
- Creates parent directories automatically
- Fails if file already exists (use file_write_on_operator to overwrite)
- Preserves exact content formatting (no shell escaping issues)
- Automatically tracked in file history for auditing and rollback
</behavior>

<examples>
<example>User: "Create a cron job to backup daily" → file_create_on_operator(file_path="/etc/cron.d/backup", content="0 2 * * * /usr/local/bin/backup.sh", justification="Create daily backup cron job")</example>
<example>User: "Add a new nginx config" → file_create_on_operator(file_path="/etc/nginx/conf.d/new-site.conf", content="server { listen 80; ... }", justification="Create nginx config for new site")</example>
</examples>
