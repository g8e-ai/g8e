Revoke an AWS capability that was previously granted through the g8e platform.

<parameters>
<required>
- intent_name: The AWS capability to revoke (ec2_discovery, ec2_management, s3_read, s3_write, terraform_state, cloudwatch_logs, secrets_read, etc.)
- justification: Why this capability is being revoked
</required>
</parameters>

<behavior>
- Detaches the AWS IAM policy from the Operator Role
- System handles the IAM policy detachment automatically via the Escalation Role
- After capability is revoked, confirm to the user that the permission has been removed
- Use this when the user asks to remove a previously granted permission
</behavior>

<examples>
<example>User: "Revoke my S3 write permissions" → revoke_intent_permission(intent_name="s3_write", justification="Task completed, revoking write access")</example>
<example>User: "Remove EC2 management access" → revoke_intent_permission(intent_name="ec2_management", justification="Cleanup of temporary permissions")</example>
</examples>
